## WHAT THIS PROJECT IS

MedConnect India is a Next.js web platform connecting India's pharma ecosystem:
MRs (medical representatives), doctors, clinic staff, chemists, stockists, and
pharma companies. A trust network — every feature must earn trust: secure,
reliable, efficient, no shortcuts. Current data is placeholder; structure first.

Core promise: "Find doctors in real time, discover jobs, track visits, and grow
your pharma career — all in one place."
Positioning: "India's Pharma Ecosystem, Connected."

Intermediate coder. 2–4 hrs/day alongside day job.

(Prisma enum: ADMIN, MEDICAL_REP, DOCTOR, CLINIC_STAFF, CHEMIST, STOCKIST, PHARMA_COMPANY)

- call_mr: admin, doctor, chemist, stockist

rest → /dashboard. Self-signup (/signup) creates MR accounts ONLY; admin creates
all other roles. Doctor accounts attach to directory profiles via one-time
atomic claim flow (POST /api/doctors/claim).

## ENGINEERING PRINCIPLES (AGENTS.md — non-negotiable, all helpers exist)

1. Roles/permissions ONLY in lib/roles.ts (rolesWith) — never hard-code.
2. Identity from session (requireUser in lib/authz.ts), NEVER from request body.
   Ownership checks gate personal resources; admin may override.
3. Every route wrapped in guarded() (lib/api.ts) → clean JSON 500s.
   Every write endpoint rate-limited (lib/rate-limit.ts) by user id or IP.
4. Every data edit audited via DoctorUpdate (who/what/when) — no anonymous
   changes ever. Surface freshness + editor in UI ("12m ago by Clinic Staff").
5. Clamp all free-text (trim + length cap) before storing.
6. Concurrency-sensitive writes atomic (updateMany guards, unique constraints).
7. IST calendar days ("YYYY-MM-DD") via lib/ist.ts only.
8. snake_case API JSON via lib/serialize.ts only.
9. bcrypt passwords · security headers in next.config.ts (nosniff, DENY frames,
   referrer policy, permissions policy, no x-powered-by).
10. Boring proven patterns. Small verifiable steps. npx tsc --noEmit before "done".
